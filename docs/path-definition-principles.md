# Path Definition Principles

## Purpose
Each path is not a service, but a way of engaging with the same question.

---

## Structure

Each path must follow this structure:

1. Where the question appears
2. How we relate to it
3. What begins to move

---

## Language Rules

- Use only affirmative expressions
- Avoid explanation or justification
- Avoid service language (支援, 提供, ソリューション)
- Prefer presence over instruction

---

## Tone

- quiet
- grounded
- non-persuasive
- observational

---

## Goal

The reader should not feel:
"I should choose this"

But:
"This is where I already am"
