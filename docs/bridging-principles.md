# Bridging Principles — Philosophy to Reality

## Purpose
The site is built on philosophy, but must still connect to lived experience.

The goal is not to explain, but to allow the reader to recognize themselves.

---

## Core Principle

Do not add explanations.
Add minimal bridges that allow meaning to emerge naturally.

---

## Bridge Definition

A bridge is:
- One line only
- Not explanatory
- Not persuasive
- Not instructional
- Not structured like a service description

It should feel like:
"Ah, this might be about me."

---

## Placement Rules

### 1. After philosophy (Hero → Body)
Insert one soft line that connects inner language to lived experience.

Example tone:
- "その感覚は、形を変えて現れることがあります。"

---

### 2. Before service sections (e.g. THREE PATHS)
Insert one line that connects the question to multiple approaches.

Example tone:
- "同じ問いでも、辿り方はそれぞれにあります。"

---

## Language Constraints

Avoid:
- "アプローチ"
- "ソリューション"
- "サービス説明"
- Direct explanations

Prefer:
- ambiguity
- softness
- recognition over instruction

---

## Intensity Control

Bridges must remain:
- 30–40% strength of normal explanatory language
- softer than body text
- never dominant

---

## Success Criteria

A correct bridge:
- is almost unnoticed
- removes the gap between philosophy and action
- does not feel like a transition
- does not feel written for conversion

---

## Failure Signs

If any of these appear, remove or rewrite:
- feels like marketing
- feels like explanation
- feels like guiding the reader
- feels like labeling the paths

---

## Final Note

The site works because it does not explain.

Bridges must preserve that silence while allowing entry.
